# medical-portal-network-traffic-analysis
This project demonstrates network-level analysis of a Spring Boot-based Medical Portal application using Wireshark. The goal was to observe and understand how HTTP traffic behaves during authentication and API communication, including TCP handshake, request/response flow, and status codes.

---

## Objectives
- Understand how client-server communication works over TCP/IP
- Capture and analyze HTTP requests generated by a Spring Boot backend
- Observe authentication flow using `POST /login`
- Identify how HTTP requests and responses are structured at the packet level

---

## Tech Stack
- Java (Spring Boot)
- IntelliJ IDEA
- Wireshark
- HTTP / TCP/IP networking
- curl / browser for request generation

---

## System Architecture
Client (Browser / curl) → Spring Boot Backend (localhost:8080) → Response  
Traffic was captured and analyzed using Wireshark on the loopback interface.

---

## Key Observations

### 1. TCP 3-Way Handshake
Before any HTTP communication, a TCP connection is established between the client and server:

- SYN
- SYN-ACK
- ACK

This ensures reliable communication over the network.

---

### 2. HTTP GET Request
The application generates HTTP GET requests when accessing endpoints such as:

- `/login`

This request is used to load the login page from the server.

---

### 3. Authentication Flow (POST Request)
The most important part of the analysis:

- `POST /login` sends user credentials to the server
- The server processes authentication logic
- The response returned is:
  - `200 OK` for successful authentication
  - `401 Unauthorized` for failed authentication

This demonstrates a basic authentication workflow in a REST API.

## Key Takeaways
- HTTP requests are visible in plaintext when not encrypted
- TCP establishes a reliable connection before application data is exchanged
- Authentication flows can be directly observed at the packet level
- Wireshark provides visibility into backend request/response behavior
